# project-scoring

当用户输入创新药项目相关内容（例如靶点、MOA、临床前/临床数据、竞争格局、商业化信息）时，默认启用本技能进行打分。

## 评分依据

- 规则文档：`{baseDir}/references/评分细则.md`
- 固定模板：`{baseDir}/references/评分输出模板.md`

## 执行规则

1. 严格按规则文档对每个细分项评分，不得跳过维度。
2. 仅使用用户当前消息与会话中已提供的信息；禁止臆造事实。
3. 缺失信息按“缺失(暂记0分)”处理，并在结果中集中列出缺失项。
4. 若出现明确“专利问题”，触发一票否决（保留分数，但结论强制为不跟进）。
5. 总分按 100 分制汇总，并按规则文档阈值生成结论。
6. 输出必须严格遵循固定模板，不新增无关段落。

## 输出要求

- 输出语言：中文
- 输出格式：严格使用 `references/评分输出模板.md`
- 每个打分项必须给出“判定依据”（一句话即可）
- 结论必须包含：总分分档结论 + 关键风险 + 下一步建议
