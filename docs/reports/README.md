## 📈 报告类型说明

### 审查报告 (Review Reports)
记录代码审查、架构审查等活动的结果和建议。

### 总结报告 (Summary Reports)
对某个阶段或特定主题的工作进行总结，包括测试、开发、部署等。

### 改进报告 (Improvement Reports)
记录改进项目的计划、执行和效果，便于跟踪和回顾。

### 验证报告 (Verification Reports)
记录功能验证、性能测试、安全审计等验证活动。

### 变更日志 (Changelogs)
记录设计、架构、API 等方面的变更历史。

## 🔍 如何使用这些报告

1. **项目回顾**：了解项目的改进历程和决策过程
2. **问题追溯**：查找历史问题及其解决方案
3. **经验借鉴**：学习过去的成功经验和失败教训
4. **新成员入门**：快速了解项目演进历史
