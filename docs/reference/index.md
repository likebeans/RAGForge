# 参考资料

本节包含 Self-RAG Pipeline 项目的参考资料、历史文档、中文内容和各种模板。这些资料为开发者和用户提供深入的技术细节和历史背景。

## 概述

参考资料部分组织为以下几个主要类别：

- **📋 变更日志**：记录项目重要变更和版本历史
- **🇨🇳 中文内容**：保留的中文技术文档和指南
- **📚 历史文档**：项目开发过程中的历史记录
- **📝 模板**：文档和开发模板

## 内容导航

### 📋 项目变更

- **[变更日志](./changelog)** - 文档重组和项目变更记录

### 🇨🇳 中文内容保留区

中文内容区保留了项目中有价值的中文技术文档，这些文档提供了独特的技术见解和本地化内容：

- **[中文内容索引](./chinese/)** - 所有中文文档的导航
- **[开发指南](./chinese/development-guide)** - 详细的中文开发指南
- **[优化指南](./chinese/optimization-guide)** - 性能优化和算法调优
- **[实践总结](./chinese/practice-summary)** - 项目实践经验总结

**为什么保留中文内容？**

1. **技术深度**：中文文档包含了详细的技术实现细节
2. **本地化体验**：为中文用户提供更好的阅读体验
3. **历史价值**：记录了项目开发过程中的重要技术决策
4. **实践经验**：包含了大量实际部署和使用经验

### 📚 历史文档归档

历史文档区保存了项目开发过程中的重要记录，帮助理解项目演进历程：

- **[历史索引](./history/)** - 历史文档导航和时间线
- **[第一阶段开发](./history/phase1-development)** - 项目初期开发记录
- **[第二阶段开发](./history/phase2-development)** - 功能扩展阶段
- **[第三阶段测试](./history/phase3-development)** - 测试和优化阶段

**历史文档的价值：**

1. **架构演进**：了解系统架构的演进过程
2. **决策背景**：理解重要技术决策的背景和原因
3. **经验教训**：学习开发过程中的经验和教训
4. **问题解决**：参考历史问题的解决方案

## 维护说明

### 内容组织原则

1. **语言分离**：中文和英文内容分别组织
2. **时间归档**：历史文档按时间和阶段归档
3. **主题分类**：相关内容按主题分组
4. **交叉引用**：相关文档之间建立链接

### 更新频率

- **变更日志**：随项目变更实时更新
- **中文内容**：根据需要定期更新
- **历史文档**：保持稳定，仅修复错误
- **模板**：根据流程变化更新

### 质量控制

1. **内容审查**：定期审查内容的准确性和时效性
2. **链接检查**：定期检查内部和外部链接
3. **格式统一**：保持文档格式的一致性
4. **索引维护**：及时更新各种索引和导航
